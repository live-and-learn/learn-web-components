Web Components: Lesson 1 Exercises

Introducing the HTML <video> tag.

With HTML5 we already have some 'Web Components' in the form of some new complex HTML tags.

The <video> tag looks simple like other HTML tags and yet is able to display a video file. If you right click on the video, there are some limited playback controls. With the addition of a simple attribute you can add a playback controls bar to the bottom of the video.

How does this work?

When you use the <video> tag the web browser 'knows' how to add to make the tag look and work like a video player. And there are actually other HTML elements 'hidden' within the <video> tag. This is one of 4 ideas that groups are trying to standardizeas part of the Web Components standards. This 'hidden' area of a Web Component is known as the "Shadow DOM" and is like the DOM representation of a web page, but just for a specific control like the <video> tag.

Now let's take a look at the <video> tag. Open this file in Google Chrome. Notice the simplicity of the single <video> tag. Right now it looks like it is impossible to play the video because the playback controls are missing. You will add the controls in the next lesson. For now you can right click on the video and choose the option to show the controls.


        <video src="./welcome.mov" type="video/mov"></video>

Do Exercise 2

Experiment with the <video> tag by adding the "controls" attribute to it. After you have added this attribute, refresh the page in you browser to see the change.

Try adding some of these attributes to the<video> tag:

  • autoplay
  • controls
  • height="" and width=""
  • loop
  • muted
  • poster="url"
  • preload="auto|metadata|none"
  • src="url"
  • Global and Event attributes

There are also other tags or text that you can nest in the video tag.

  • source as in 
    <source src="movie.mp4" type="video/mp4">
  • plain text that will be displayed if the browser does not support the video tag 
    Your browser does not support the video tag.

Other Info

You may be wondering where the markup is for video controls and other attributes that you add or change. These are hidden in within the Web Component in an area known as the Shadow DOM. There will be more about this later.
